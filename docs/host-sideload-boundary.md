# 宿主侧载与资源读取边界

本页定义 UnicodeArtJs 各宿主消费声明式资源包、静态画廊资产和本地用户文件时的共同边界。它补充 [宿主接入指南](host-integration.md)、[声明式扩展 SDK](extension-sdk.md) 和 [桌面宿主基线](desktop-host-baseline.md)，重点回答一个问题：宿主可以读取什么，何时读取，以及哪些行为必须留到新格式或独立权限模型。

## 基本原则

- Core 不访问文件系统、不扫描目录、不联网、不执行扩展代码。
- UAEM v1 是本地声明式资源包，不是脚本插件、安装包、远程市场或自动更新机制。
- 有文件系统权限的宿主只能读取用户显式选择的清单，以及清单 `resources[]` 中声明的本地资源。
- 宿主只读取用户显式选择的清单，不扫描目录、不执行扩展代码、不自动联网。
- 宿主只读取 `resources[]` 中声明的文件，不扫描同级目录，也不发现隐藏入口。
- Web 默认只能做单文件清单检查；静态画廊只能读取同源、随站发布、已审核的 JSON。
- VS Code 和 Desktop 后续可提供资源包管理 UI，但必须保留“用户选择、Core 校验、真实路径复核、资源逐项解析”的顺序。
- Compatible 应用可以扩展宿主能力，但不能把额外 runtime、decoder、远程服务或宽松度较低的组件反向混入 Clean 默认路径。

## 宿主读取矩阵

| 宿主 | 清单读取 | 资源读取 | 当前推荐用途 | 禁止行为 |
| --- | --- | --- | --- | --- |
| Core | 只接收调用方传入的 JSON 文本 | 不读取文件 | 解析 UAEM、UAF、语义文档并评估兼容性 | 访问文件系统、网络、shell、宿主窗口或框架对象 |
| CLI | 用户传入本地 `unicode-art-extension.json` 路径 | 读取同一根目录内 `resources[]` 显式声明的文件 | 作者自测、CI 预检、发布前验证 | 扫描目录、安装扩展、写全局状态、执行扩展代码 |
| Web | 用户选择单个清单 JSON 或同源画廊索引 | 默认不读取相邻资源；只读取同源 `gallery/` 内审核资源 | 浏览器兼容性预览、静态画廊展示、source-first 编辑 | 自动读目录、任意 URL 下载、加载用户提供脚本或字体 URL |
| VS Code Extension | 后续通过命令或 WebView 选择清单 | 后续只在用户确认后读取声明资源 | 编辑器内模板、字体包和本地资源包管理 | 把资源包加入自动激活项、绕过 WebView 协议、执行清单外文件 |
| Desktop | 通过打开对话框选择清单或项目 | 通过受控 IPC/command 读取声明资源 | 本地创作工作台、项目文件、可控缓存 | 向 renderer 暴露通用文件系统、shell、目录扫描或自动联网 |
| Compatible 应用 | 按自身权限模型显式选择 | 可加入额外 adapter 或缓存，但需公开说明 | 独立应用、兼容格式、平台集成 | 混入默认 Core/CLI/Web/VS Code lockfile，弱化许可证材料 |

## UAEM v1 侧载流程

有文件系统权限的宿主应采用以下顺序：

1. 用户或调用方显式提供清单文件。
2. 宿主读取清单文本，并调用 Core 解析与兼容性评估。
3. 若目标宿主、Core 版本或能力不兼容，停止读取资源并给出机器可读原因。
4. 宿主把清单所在目录作为扩展根目录。
5. 对每个 `resources[]` 条目，先由 Core 校验相对 POSIX 路径和资源种类。
6. 读取前使用真实路径或等价机制确认资源仍位于扩展根目录内，防止符号链接或宿主路径语义逃逸。
7. 只读取被声明的资源文件，不扫描目录、不加载额外清单、不解析隐藏入口。
8. 将资源文本交给对应 Core 解析器；失败时不得替换当前可用工作区。
9. 宿主可以缓存解析后的 canonical JSON、资源摘要或最近选择记录，但不得把它变成自动安装或后台更新。

无文件系统目录权限的宿主，例如普通浏览器页面，只能执行前半段：读取用户选择的清单文本、调用 Core 解析并展示兼容性结果。若未来使用 File System Access API 或打包环境提供目录授权，也必须走显式目录选择和逐项资源读取，不得默认扩大 UAEM v1 能力。

## 静态画廊边界

静态画廊不是 UAEM 资源包市场。它是随 GitHub Pages 发布的受控示例目录：

- 索引格式为 `unicode-art-gallery-index@1`。
- 资源必须位于同源 `gallery/artworks/` 下。
- 资源内容只允许已审核的 `.uadoc.json` 或 `.uafont.json`。
- 浏览器通过 `gallery-index.js` 校验索引、路径和双语元数据，再把资源文本交给 Core 校验。
- 画廊不接受用户上传，不自动同步远程仓库，不加载第三方脚本，也不执行资源包逻辑。

投稿、审核和回退流程见 [静态画廊投稿指南](gallery-submission.md) 与 [静态画廊审核指南](gallery-review.md)。

## 错误、缓存与回退

宿主应保持失败可恢复：

- 清单解析失败、兼容性失败、资源读取失败或资源解析失败时，不替换当前工作区。
- 错误提示优先使用 Core `code`、`messageKey`、`messageParams` 和宿主自己的错误域。
- 缓存应记录来源、Core 版本、资源摘要和用户授权状态；不要默默重新读取历史外部路径。
- 删除、禁用或回退资源包时，应移除索引或缓存引用，不需要修改 Core 数据结构。

## 未来扩展

动态插件、签名资源包、远程索引、扩展市场、自动更新、联网下载和可执行能力都不属于 UAEM v1。它们需要新的格式版本、权限模型、签名策略、内容审核和 Compatible 许可证材料。任何宿主都不应用“兼容 UAEM v1”来暗示已经支持这些未来能力。
