# 静态画廊审核指南

本指南面向维护者和协作者，用于审核进入 UnicodeArtJs 静态作品画廊的 JSON 资产。画廊是 GitHub Pages 上的受控静态示例集，不是开放上传平台。

## 审核范围

当前默认画廊只接收两类声明式资产：

- `semantic-document@1`：语义布局文档，文件名以 `.uadoc.json` 结尾。
- `unicode-art-font@1`：UAF 艺术字字体，文件名以 `.uafont.json` 结尾。

作品应优先为原创内容。第三方来源即使采用宽松许可，也需要完整出处、许可证和可复核归属；来源不清、复制 GPL/专有字体或依赖在线生成器结果的作品不进入默认画廊。

## 审核流程

1. 确认投稿说明与作品文件一致：作品类型、作者、标题、说明、标签和许可证不能互相矛盾。
2. 检查 JSON 文件只位于 `packages/web/public/gallery/artworks/`，路径为小写稳定文件名，不包含脚本、远程 URL、本机绝对路径、账号、密钥或外部依赖。
3. 检查 `packages/web/public/gallery/index.json` 中的 `id`、`source`、`kind`、`license`、`reviewedAt` 与作品文件一致。
4. 运行自动验证命令，至少包括 `npm run gallery:check` 与 `npm --workspace packages/web test`。
5. 在本地或 GitHub Pages 预览作品，确认筛选、预览、复制、下载和“在编辑器中打开”没有明显异常。
6. 合并后观察 GitHub Actions 与 Pages 部署结果，确认线上画廊可加载新索引。

## 许可确认

审核时不要只看许可证字符串，还要看作品来源是否可信：

- 原创作品应在 Issue、PR 或提交说明中明确写明原创。
- 第三方素材必须提供出处、许可证版本和再分发条件。
- AI 辅助生成的作品应确认没有复制受限第三方字形、图片或网站输出。
- 如果许可证或来源解释不充分，应要求作者补充，而不是先合并再补。

默认画廊允许的许可证范围由 `scripts/check-gallery.cjs` 中的 allowlist 管理。扩展 allowlist 前应单独评估是否仍符合项目的宽松许可证口径。

## 回退流程

如果作品上线后发现来源、许可证、渲染或内容问题，优先采用小而清晰的回退：

1. 从 `packages/web/public/gallery/index.json` 移除对应条目。
2. 删除或保留作品源文件取决于问题类型；来源或许可不清时应删除，纯渲染问题可保留在后续修复分支中。
3. 运行 `npm run gallery:check`、`npm --workspace packages/web test`，必要时运行 Web E2E。
4. 在 PR 或提交说明中写明回退原因，避免同一资产在未处理问题前重新进入画廊。
5. 发布后复核 GitHub Pages，确认画廊索引不再引用已回退资产。

## 人工预览要点

- 标题和说明在中文、英文界面中都自然可读。
- 标签有助于筛选，不把所有作品都塞进同一个标签。
- 字素宽度、边框、表格和原字输出没有明显错位。
- 作品在常见等宽字体下仍可理解；如果依赖特定字体，应在说明中避免夸大兼容性。
- 预览内容适合作为公开示例，不包含内部开发过程、临时路径或私人信息。
