# UnicodeArtJs Web 用户指南

## 快速开始

1. 打开网页
2. 在右上角选择界面语言和主题
3. 选择工作模式：**图片转字符画**、**文字Banner**，或 experimental 的 **Studio 实验**
4. 上传图片或输入文字
5. 调整参数
6. 查看预览结果
7. 导出或复制

## 功能详解

### 图片转字符画

- **上传**: 拖拽图片或点击上传区域
- **支持格式**: JPG/PNG/GIF/WebP
- **大小限制**: 最大10MB
- **参数**: 高度/宽度/字符集/字体/矩阵大小等

### 文字Banner

- **输入**: 在下方的文本框中输入文字
- **实时预览**: 输入后自动生成字符画
- **参数**: 高度/宽度/字符集/字体等

### 字符集

| 字符集 | 说明 |
|--------|------|
| ASCII | 英文 + 符号 (95个可打印字符) |
| 扩展字符 | 更多符号和装饰字符 |
| 简体中文 | GB2312常用汉字 |
| 自定义 | 手动输入字符（从暗到亮排列） |

### 字体

- **视觉字体**: 用于渲染输入文本（采样用）
- **字素字体**: 用于预览区显示字符画（等宽字体）
- **视觉字体内边距/收缩**: 用于手动修正部分视觉字体生成中间图像时的边缘留白
- **字素宽度规则（实验）**: 用于后续字体宽度 profile 和自定义宽字符正则；当前主要作为配置入口保留

建议优先选择 Sarasa Mono SC、LXGW WenKai Mono、Source Code Pro、Liberation Mono 等开源等宽或混合等宽字体。

### 裱框

- **样式**: 单线/双线/圆角/粗线/经典/ASCII等
- **配置**: 内边距、外边距、标题、阴影

### Studio 实验

**Studio 实验**用于在浏览器中试用 source-first 的 UnicodeArt 创作工作台。它可以编辑布局文档、UAF 艺术字字体和 CellCanvas 字素画，并提供本地校验、预览、模板、JSON 导入导出和基础投影导出。

需要注意：

- 该入口仍是 experimental，不代表 Studio 已经 stable。
- 模板和草稿只保存在当前浏览器的 `localStorage`，不会自动上传。
- 导入 JSON 会先经过 Core 校验，失败时不会覆盖当前内容。
- `.uart-project.json` / `studio-project@0` 目前只是内部项目包络，后续仍可能调整。
- 资源入口只读取本站同源、已审核的资源清单，并要求手动确认后才导入编辑器。
- AI 提案预览使用确定性 mock provider；不会联网、不会读取源码全文，也不会直接写入项目。
- 字体显示依赖浏览器和本机字体。缺少所选字素字体时，预览和导出会按浏览器规则回退。

### 主题

- **默认**: 亮色简约
- **暗黑**: 深色护眼
- **高对比度**: 黑白分明
- **Solarized浅色**: 温暖柔和
- **Nord极冰**: 冷色调

### 导出

- **TXT**: 纯文本格式
- **HTML**: 带样式的网页
- **PNG**: 图片格式
- **复制**: 直接复制到剪贴板

### 语言

右上角语言选择会同时影响：

- 页面主要按钮、标签、提示和错误信息
- Core 的 `locale` 配置
- HTML 导出的 `lang` 属性

设置会保存在浏览器本地存储中。

## 键盘快捷键

- `Tab` 切换到下一个输入
- `Enter` 在文字模式中换行

## 常见问题

**Q**: 文字转字符画时提示"生成失败"
**A**: 请检查Core库是否正确加载（查看浏览器控制台是否报错）

**Q**: PNG导出效果不理想

**A**: PNG 使用 Canvas 渲染，效果取决于浏览器和系统字体。请优先选择开源等宽或混合等宽字体。

**Q**: 设置保存了吗？

**A**: 所有参数自动保存在浏览器本地存储中。

**Q**: 为什么不同浏览器或不同字体下字符画略有差异？

**A**: 浏览器 Canvas 和字体渲染引擎会影响文字采样和导出。当前以 Chrome 120+ 为主要验证基线，跨浏览器像素级完全一致暂不作为承诺。
