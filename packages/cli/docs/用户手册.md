## 🚀 快速开始

### 安装

```bash
# 全局安装（推荐）
npm install -g unicode-art-cli

# 或本地安装
npm install unicode-art-cli
```

### 基本用法

```bash
# 文本转字符画
unicode-art text "Hello World"

# 图片转字符画
unicode-art image photo.jpg
```

## 💻 命令参考

### 通用选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| `-c, --config <path>` | 配置文件路径 | 自动查找 |

### `image` 命令 - 图片转字符画

```bash
unicode-art image <input> [options]
```

#### 参数

- `<input>` - 输入图片路径（必需）

#### 选项

**输出控制**:
- `-o, --output <path>` - 输出文件路径
- `-p, --print` - 直接打印到终端（默认: true）

**尺寸参数**:
- `-e, --height <number>` - 输出高度（行数）
- `-w, --width <number>` - 输出宽度（列数）

**字符集**:
- `-a, --chars <string>` - 自定义字符集
- `--charset <type>` - 预设字符集 (ASCII\|EXTENDED\|CHINESE_SIMPLE)

**字体配置**:
- `-f, --font <name>` - 字体名称或路径
- `--font-style <style>` - 字体样式 (normal\|bold\|italic)

**算法配置**:
- `-m, --matrix <size>` - 矩阵大小（默认: 6）
- `-r, --ratio <number>` - 垂直水平比例（默认: 2.0）
- `--interpolation <type>` - 插值算法 (nearest\|bilinear\|bicubic)
- `--wide-char-ratio <number>` - 宽字符匹配阈值（默认: 1.5）

**颜色与格式**:
- `-v, --invert` - 反转颜色
- `--trim-trailing-spaces` - 去除行尾空格
- `--format <format>` - 输出格式 (plain\|html\|ansi)
- `--image-backend <backend>` - Node 图片后端 (sharp\|napi-rs)，`napi-rs` 为宽松许可证实验后端
- `-b, --box <json-or-style>` - 裱框配置，支持 `true`、`false`、内置样式名或 JSON 对象

**其他**:
- `--lang <locale>` - 语言 (zh-CN\|en-US)
- `--no-config` - 禁用配置文件自动发现

#### 示例

```bash
# 基本用法
unicode-art image photo.jpg

# 指定输出文件
unicode-art image photo.jpg -o output.txt

# 高分辨率输出
unicode-art image photo.jpg -e 50

# 使用扩展字符集
unicode-art image photo.jpg --charset EXTENDED

# HTML彩色输出
unicode-art image photo.jpg --format html -o output.html

# 为输出添加外框
unicode-art text "Hello" --box round
unicode-art text "Hello" --box "{\"style\":\"ascii\",\"padding\":1,\"title\":\"Demo\"}"
# PowerShell JSON 写法
unicode-art text "Hello" --box '{\"style\":\"ascii\",\"padding\":1,\"title\":\"Demo\"}'
# 固定宽度、换行和阴影
unicode-art text "HelloWorld" --box '{\"style\":\"ascii\",\"width\":8,\"overflow\":\"wrap\",\"shadow\":{\"style\":\"block\"}}'
# Layout stage 网格裱框
unicode-art text "AB" --box '{\"renderStage\":\"layout\",\"mode\":\"grid\",\"style\":\"ascii\",\"separators\":{\"rows\":true,\"columns\":true},\"cell\":{\"minWidth\":1,\"minHeight\":1}}'

# 反转颜色
unicode-art image photo.jpg -v

# 自定义开源字体和尺寸
unicode-art image photo.jpg --visual-font "Noto Sans SC" -e 30 -m 8
```

### `text` 命令 - 文本转字符画

```bash
unicode-art text <text> [options]
```

#### 参数

- `<text>` - 输入文本字符串（必需）

#### 选项

包含所有 `image` 命令的选项，外加：

**文本布局**:
- `--text-align <align>` - 文本对齐 (left\|center\|right)
- `--line-spacing <number>` - 行间距（默认: 1.0）
- `--height-mode <mode>` - 高度模式 (auto\|fixed)

`text -` 可以从 stdin 读取文本：

```bash
echo "Hello" | unicode-art text - --height 8
```

#### 示例

```bash
# 基本用法
unicode-art text "Hello World"

# 中文文本
unicode-art text "你好世界" --charset CHINESE_SIMPLE

# 居中对齐
unicode-art text "Center" --text-align center

# 指定字体
unicode-art text "Bold" --visual-font "Noto Sans SC" --font-style bold

# 多行文本（使用换行符）
unicode-art text "Line1\nLine2\nLine3" --line-spacing 1.5
```

## 📝 配置文件

### 支持的格式

- `.unicode-artrc.yml`
- `.unicode-artrc.json`
- `unicode-art.config.js`

### 配置优先级

```
命令行参数 > 配置文件 > 默认值
```

### 完整配置示例 (.unicode-artrc.yml)

```yaml
# 输出配置
output:
  format: plain              # plain | html | ansi
  target: terminal           # plain | terminal | web | vscode | electron | html | ansi
  trimTrailingSpaces: false

# 图片输入配置
image:
  backend: sharp             # sharp | napi-rs

# 裱框配置，可设为 false、true、样式名或对象
box:
  mode: outer                # outer | lines | cells | grid
  renderStage: post          # post | layout
  style: round               # single | double | round | bold | classic | ascii | none
  padding: 1
  width: auto                # auto 或非负整数
  height: auto               # auto 或非负整数，表示内容区行数
  verticalAlign: top         # top | middle | bottom
  overflow: expand           # expand | truncate | wrap
  shadow: false              # false | true | { style, offsetX, offsetY }
  separators:
    rows: false              # true | false | number[]
    columns: false           # true | false
  cell:
    padding: 0
    minWidth: 0
    minHeight: 0
  title:
    text: UnicodeArtJs
    align: center

# 尺寸配置
size:
  height: 20                 # 默认高度（行数）
  # width: 80                # 可选：固定宽度

# 字符集配置
charset:
  type: ASCII                # ASCII | EXTENDED | CHINESE_SIMPLE
  # customChars: " .:-=+*#%@"

# 字体配置
font:
  name: Noto Sans SC         # 字体名称或路径
  style: regular             # regular | bold | italic | bold-italic
  reduce: 0.8                # 字体缩放比例

# 算法配置
algorithm:
  matrixSize: 6              # 字符矩阵大小
  ratio: 2.0                 # 垂直/水平比例
  interpolation: bilinear    # nearest | bilinear | bicubic
  wideCharRatio: 1.5         # 宽字符匹配阈值
  enableEarlyTermination: true

# 颜色配置
color:
  invert: false              # 是否反转颜色

# 文本配置（仅text命令）
text:
  align: left                # left | center | right
  lineSpacing: 1.0           # 行间距倍数
  heightMode: line           # line | total

# 性能配置
performance:
  maxParallelTasks: 0        # 最大并行任务数（0=自动）

# 国际化
i18n:
  lang: zh-CN                # zh-CN | en-US
```

### 最小配置示例

```yaml
size:
  height: 30

charset:
  type: EXTENDED
```

## 💡 使用示例

### 场景1: 生成头像字符画

```bash
# 将头像转换为字符画并保存
unicode-art image avatar.jpg -e 40 --charset EXTENDED -o avatar.txt
```

### 场景2: 制作ASCII艺术签名

```bash
# 使用特殊字体生成签名
unicode-art text "John Doe" -f "Brush Script" -e 10 -o signature.txt
```

### 场景3: 网页嵌入字符画

```bash
# 生成HTML格式
unicode-art image logo.png --format html -o logo.html
```

然后在HTML中引用：
```html
<!DOCTYPE html>
<html>
<body>
  <pre id="art"></pre>
  <script>
    fetch('logo.html')
      .then(r => r.text())
      .then(html => {
        document.getElementById('art').innerHTML = html;
      });
  </script>
</body>
</html>
```

### 场景4: 终端美化输出

```bash
# ANSI彩色输出（如果支持）
unicode-art image banner.png --format ansi
```

### 场景5: 批量处理

```bash
# Bash脚本批量转换
for img in *.jpg; do
  unicode-art image "$img" -o "${img%.jpg}.txt"
done
```

## 🎨 字符集说明

### ASCII
基础ASCII字符集，适合英文文本和简单图像。
```
 .:-=+*#%@
```

**适用场景**: 
- 英文文本
- 简单图标
- 需要广泛兼容性的场景

### EXTENDED
扩展字符集，包含更多灰度级别和特殊符号。
```
 ░▒▓█▀▄■□○●◇◆▲▼★☆
```

**适用场景**:
- 高质量图像转换
- 需要丰富灰度层次
- 现代终端环境

### CHINESE_SIMPLE
简化中文字符集，包含3500+常用汉字。

**适用场景**:
- 中文文本转换
- 中日韩混合内容
- 需要宽字符支持的场景

## 🔧 常见问题

### Q1: canvas未安装怎么办？

**问题**: 运行text命令时提示canvas依赖缺失。

**解决**:
```bash
npm install canvas
```

或者使用预渲染的SVG字体替代方案。

---

### Q2: 如何调整输出质量？

**提高分辨率**:
```bash
# 增加行数
unicode-art image photo.jpg -e 50

# 增大矩阵尺寸
unicode-art image photo.jpg -m 8
```

**优化速度**:
```bash
# 使用最近邻插值（更快）
unicode-art image photo.jpg --interpolation nearest
```

---

### Q3: 字体找不到怎么办？

**检查系统字体**:
```bash
# macOS/Linux
fc-list | grep "FontName"

# Windows
# 在 C:\Windows\Fonts 中查找
```

**使用字体文件路径**:
```bash
unicode-art text "Hello" -f "/path/to/font.ttf"
```

---

### Q4: 如何自定义字符集？

**方法1: 命令行参数**
```bash
unicode-art image photo.jpg -a " .:-=+*#%@"
```

**方法2: 配置文件**
```yaml
charset:
  customChars: " .:-=+*#%@"
```

---

### Q5: 输出乱码怎么办？

**确保UTF-8编码**:
```bash
# 设置终端编码
export LANG=en_US.UTF-8

# 或使用--lang参数
unicode-art text "中文" --lang zh-CN
```

**检查文件编码**:
```bash
# 查看文件编码
file -I output.txt
```

---

### Q6: 性能太慢怎么办？

**优化建议**:
1. 降低输出高度（`-e 20` 而不是 `-e 50`）
2. 使用较小的矩阵尺寸（`-m 4` 而不是 `-m 8`）
3. 使用最近邻插值（`--interpolation nearest`）
4. 禁用早期终止（通过配置文件设置 `enableEarlyTermination: false`）
